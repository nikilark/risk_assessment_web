# Risk Assessment Web — посібник з використання

## Огляд

Risk Assessment Web призначений для оцінки ризику впливу забруднювальних речовин на здоров'я населення. Застосунок поєднує введення вхідних даних, розрахунок канцерогенного і неканцерогенного ризику, оцінку ризику від зважених частинок, побудову таблиць, графіків, мап і формування звіту.

Основний сценарій тепер зібраний у чотири робочі сторінки:

- **Проєкт** — назва проєкту, типи дослідження, вибір і додавання речовин.
- **Об'єкти** — додавання об'єктів з мапи або вручну, населення та концентрації для обраних речовин в одній таблиці.
- **Результати** — розрахункові таблиці, графіки та мапи в окремих згортних блоках.
- **Звіт** — впорядкування вибраних таблиць, графіків і мап та підготовка друкованої/PDF-версії документа.

Ліворуч розміщена навігаційна панель. Сторінки, до яких ще не можна перейти, позначені як неактивні. Коли наступна сторінка стає доступною, пункт меню коротко підсвічується. Внизу панелі є пункти **Налаштування** і **Довідка**.

## Встановлення

Нова версія застосунку постачається як offline-first PWA: архів містить статичний web-застосунок і локальний launcher, який відкриває його в браузері через `localhost`. Після першого запуску інтерфейс, типовий каталог, розрахунки, автозбереження, імпорт/експорт JSON, TSV/PNG/GeoJSON-експорт і звіт працюють без інтернету. Мапи, визначення назви об'єкта за координатами та шар джерел забруднення використовують онлайн-сервіси OpenStreetMap/Overpass.

Для збирання web-версії з коду потрібен Node.js 20.19+. Для launcher потрібен Go 1.22+.

```bash
make install
make audit
make build
make preview
```

Запуск через launcher після збирання:

```bash
make launcher-build
./launcher/bin/rat-launcher --dir web/dist
```

## Налаштування

Сторінка **Налаштування** відкривається знизу лівої панелі. Тут доступні:

- відкриття проєкту;
- збереження поточного проєкту у JSON;
- вибір базового або розширеного каталогу речовин;
- системна, світла або темна тема;
- основний і додатковий кольори інтерфейсу з попереднім переглядом.

Назва проєкту більше не знаходиться в налаштуваннях: вона редагується зверху на сторінці **Проєкт**.

## Сторінка "Проєкт"

У верхній частині сторінки введіть назву проєкту. Нижче увімкніть потрібні типи дослідження перемикачами. Для канцерогенного і неканцерогенного ризику спочатку вмикається загальна група, після чого доступні окремі перемикачі хронічного і гострого розрахунку:

- канцерогенний ризик: хронічний або гострий;
- неканцерогенний ризик: хронічний або гострий;
- ризик від дії зважених частинок.

Таблиця речовин одразу фільтрується за обраними типами. Пошук працює по назві речовини. Речовини залишаються в алфавітному порядку, навіть коли ви їх вибираєте.

Речовину можна додати вручну внизу сторінки. Заповнюйте лише поля, потрібні для обраних типів дослідження:

- назва;
- CAS;
- клас небезпеки;
- RFC і органи ураження для хронічного неканцерогенного впливу;
- ARFC і органи ураження для гострого неканцерогенного впливу;
- SFi для канцерогенного впливу.

Після вибору хоча б однієї речовини активується сторінка **Об'єкти**.

## Сторінка "Об'єкти"

На цій сторінці додаються об'єкти дослідження і вводяться концентрації.

Об'єкт можна додати вручну або з мапи кнопкою **Обрати на мапі**. Якщо об'єкт вибраний з мапи, координати зберігаються разом із ним і можуть використовуватися на сторінці результатів. Застосунок намагається автоматично підставити назву об'єкта за координатами; якщо онлайн-геокодування недоступне, буде використана назва з координатами.

На мапі є перемикач **Джерела забруднення**. Він показує лише довідковий візуальний шар імовірних промислових об'єктів із OpenStreetMap/Overpass у поточній області мапи. Цей шар не бере участі в розрахунках і не замінює офіційні реєстри джерел викидів.

Поля об'єкта:

- місце;
- населення, якщо воно потрібне для обраного типу розрахунку;
- випадки смерті, якщо досліджуються зважені частинки.

Нижче розташована єдина таблиця об'єктів. Перші стовпці містять назву, населення і, для зважених частинок, випадки смерті; наступні стовпці відповідають обраним речовинам. Введіть концентрацію в рядку потрібного об'єкта. Сторінка **Результати** активується, коли є хоча б один об'єкт і хоча б одне значення концентрації більше нуля.

## Сторінка "Результати"

Результати згруповані в дереві ліворуч за категоріями: канцерогенний ризик, неканцерогенний ризик і зважені частинки. Виберіть потрібний результат, а справа відкрийте потрібний згортний блок:

- **Таблиця** — копіювання і збереження TSV;
- **Графік** — копіювання або збереження PNG;
- **Мапа** — копіювання PNG або даних маркерів, збереження PNG чи GeoJSON, якщо об'єкти мають координати.

У таблицях дуже малі або дуже великі значення показуються в науковій нотації, наприклад `1e-3`. У сумарному неканцерогенному ризику загальний `HI` показаний окремим стовпцем, а `HI` для критичних органів додається окремими стовпцями.

На мапі результатів також доступний перемикач **Джерела забруднення**. Він потрібен тільки для візуального контексту поточної області мапи.

Таблицю, графік і мапу можна окремо додати до звіту, тому в документ потрапляє саме той вигляд, який ви вибрали.

## Сторінка "Звіт"

На сторінці звіту показані матеріали, додані з результатів. Їх можна:

- прибрати зі звіту;
- надрукувати або зберегти як PDF через браузер.

Звіт формується за публікаційним шаблоном. Додані таблиці, графіки й мапи мають окремі заголовки, щоб було зрозуміло, який саме матеріал додано зі сторінки результатів. Графіки й таблиці у звіті залишаються світлими навіть у темній темі інтерфейсу, бо документ призначений для друку або PDF.

Базова структура звіту:

- короткий виклад;
- методика;
- вхідні дані;
- результати;
- інтерпретація;
- обмеження та застереження;
- методичні орієнтири.

Навіть якщо графіки або мапи ще не додані, документ містить структуру, вхідні таблиці та короткий опис розрахунку. Для фінальної публікації додайте ключові таблиці, графіки або мапи зі сторінки **Результати**.

## Збереження і відновлення

Поточний стан дослідження автозберігається в IndexedDB браузера. Для резервного копіювання та перенесення між комп'ютерами використовуйте JSON-експорт на сторінці **Налаштування**. Нова web-версія підтримує лише проєкти зі `schema_version: 1`; старі файли попередньої desktop-версії не імпортуються як проєкти.

## Поради з перевірки даних

- Перевіряйте одиниці вимірювання концентрацій перед введенням.
- Для порівняння об'єктів використовуйте однакові показники: ICR для канцерогенного ризику, HQ/HI для неканцерогенного ризику, AM для зважених частинок.
- Для мап обов'язково задавайте координати об'єктів.
- Не використовуйте шар **Джерела забруднення** як доказ наявності або відсутності офіційного джерела викидів: це тільки візуальна довідка за відкритими картографічними даними.
- Якщо результат виглядає аномально, спочатку перевірте концентрацію, населення, токсикологічні параметри речовини та обраний тип дослідження.

## Публікація на GitHub Pages

Для публікації web-версії в репозиторії є GitHub Actions workflow `.github/workflows/pages.yml`. Він збирає `web/dist` і публікує його на GitHub Pages.

У налаштуваннях репозиторію виберіть:

```text
Settings -> Pages -> Build and deployment -> Source -> GitHub Actions
```

Звичайна локальна збірка за замовчуванням розрахована на доменний корінь (`/`). Для GitHub Pages workflow задає `VITE_BASE_PATH=/risk_assessment_web/`, щоб застосунок працював за адресою виду `https://username.github.io/risk_assessment_web/`.
